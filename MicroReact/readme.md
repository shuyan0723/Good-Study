## Iframe (传统嵌入式方式)

### 背景
最早期的微前端架构是基于**iframe**的方式来实现的。iframe 的技术本质上允许在一个页面内嵌入另一个独立的网页，从而实现多个独立的应用的展示。

### 使用场景
这种方式常常被用于将外部的应用嵌入到当前页面中，例如在传统的企业级系统中，可能会将不同的业务模块嵌入到同一个页面内，从而形成一个"拼接"的效果。

### 优点
- 独立性：每个子应用都有自己的生命周期和上下文，彼此之间不会互相影响。
- 简单实现：实现简单，基本的HTML 页面即可使用iframe 嵌入。
### 缺点
- 性能问题：iframe 本身会产生较大的性能开销，尤其是在存在多个嵌套应用时，容易导致页面加载变慢。
- 隔离性差：由于嵌入的页面是完全独立的，无法轻松实现应用间的通信和共享状态。
- 跨域问题：iframe中得到的内容和主页面通常会存在跨域问题，这增加了开发和运维的复杂性。

### iframe

多个子应用接入到一个页面中，靠的的 iframe src
#### 怎么用？
1. 主应用：作为宿主页面，承接所有子应用
2. 子应用：每个子应用都运行在自己的iframe中，通常是独立的前端应用，他们和主应用相互独立，运行在各自的浏览器上下文中。
3. 通信：不同的iframe应用之间可以通过postMessage实现一定的通信，主应用和子应用也可以通过类型的方式进行交互。
